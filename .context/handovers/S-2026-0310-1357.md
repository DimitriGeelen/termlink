## Decisions Made This Session

1. **Three-phase Terminal.app cleanup over direct `close window id`**
   - Why: `close window id N` silently fails for busy windows. `do script "exit"` works because Terminal.app is configured to close on shell exit.
   - Alternatives rejected: System Events UI scripting (requires accessibility permissions), `close window id N saving no` (silently fails)

2. **Input validation (not sanitization) for executor.rs**
   - Why: `command.execute` RPC is designed for running shell commands — sanitizing metacharacters would break the API. Validation (null bytes, length, empty) catches abuse without limiting legitimate use.
   - Alternatives rejected: Full shell escaping of command strings (breaks intentional shell features like pipes and redirects)

3. **Exit code capture pattern for watcher scripts**
   - Why: `|| echo` swallows non-zero exits, making all commands appear successful. Capturing exit code via `CLAUDE_EXIT=0; cmd || CLAUDE_EXIT=$?` preserves the error while preventing script termination.
   - Alternatives rejected: `set -e` (too aggressive — kills watcher on any failure)

## Things Tried That Failed

1. **`close window id N saving no`** — Terminal.app accepts the command (exit code 0) but does not actually close busy windows. Silently fails.
2. **System Events UI scripting** — Blocked by accessibility permissions (`osascript is not allowed assistive access`).
3. **PID-based process cleanup** — PIDS_FILE was never populated by spawn_tracked(). Switched to TTY-based process discovery.
4. **`grep -v -E '(login|zsh|bash)'` for process filtering** — Excluded ALL bash processes including the watcher itself. Fixed by only matching login shells: `(login|-zsh|-bash)`.
