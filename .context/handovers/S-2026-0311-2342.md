## Decisions Made This Session

1. **JSONL transcript is the conversation capture format**
   - Why: Mirrors public Claude API (stable), already on disk, structured, trivial cost
   - Rejected: TermLink raw output (too noisy), Anthropic hook wait (not available today)

2. **Topic boundary = 5-minute timestamp gap, fallback to last 20 human exchanges**
   - Why: Simple, deterministic, works without ML
   - Rejected: Semantic similarity detection (too complex)

3. **Sub-agent transcripts separated to T-107**
   - Why: Different storage path (/tmp), different lifecycle, different solution
   - Rejected: Including in T-101 scope (would bloat the inception)

4. **Capture is permanent, reporting is iterative (T-104 principle)**
   - Why: Don't pre-optimize reports for questions you haven't asked yet
   - Applied to: T-104 scope (data layer first, T-105 reporting page later)

5. **Framework PR tasks are separate from build tasks, sequenced by dependency**
   - Why: Clean separation of "implement here" vs "propagate to framework"
   - Structural nudge: build template now has framework PR AC checkbox

6. **One task per framework PR, with pickup prompt written to docs/framework-agent-pickups/**
   - Why: Persistence, traceability, copy-paste to framework agent

7. **Agent Mesh (TermLink) does NOT fully solve conversation loss**
   - Why: Human↔agent conversation still lives in Claude Code context even with Agent Mesh
   - Agent Mesh helps agent↔agent but not human↔agent
