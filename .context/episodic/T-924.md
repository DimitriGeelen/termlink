# T-924: Shared TargetOpts + call_session CLI helper for cross-host routing

**Completed:** 2026-04-11
**Type:** build
**Duration:** 2026-04-11 (same day, ~35 min)

## What Was Done
Created `crates/termlink-cli/src/target.rs` with `TargetOpts` (clap::Args-derived struct with target/secret-file/secret/scope/session fields) and `call_session(opts, method, params)` async helper that routes through `connect_remote_hub` when `--target` is set, or local `client::rpc_call` when not. Includes `resolve_secret()` with precedence chain (explicit hex > secret-file > ~/.termlink/secrets/<host>.hex) and `default_scope_for(method)` for least-privilege scope defaults. 20 unit tests cover all validation paths. Module marked `#![allow(dead_code)]` until consumer tasks T-925..T-935 wire it in.

## Key Decisions
- Placed `target.rs` at crate root (alongside `cli.rs`, `config.rs`) rather than under `commands/` since it is shared infrastructure, not a command handler.
- Used `#![allow(dead_code)]` until consumer tasks wire in, to avoid warnings from unused helper code.
- `call_session` picks minimum scope per method via `default_scope_for()` table rather than always sending the strongest scope, preserving least-privilege.

## Patterns Observed
- This is the shared infrastructure prereq pattern: build the helper first with full tests, then roll out per-command in subsequent tasks (T-925..T-935).
