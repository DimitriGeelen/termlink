# T-058: Pre-Push Hook Root Cause Analysis

## Symptom

`git push` from the termlink project is blocked by audit FAILURES that belong to the **framework repo**, not the termlink project.

Observed failures:
- D2: "Human review queue — 7 tasks waiting >72h" (framework repo's tasks)
- D8: "Handover has 163 [TODO] sections" (framework repo's LATEST.md)

## Root Cause Chain

### Link 1: Pre-push hook resolves audit script via `framework_path`

```bash
# .git/hooks/pre-push (lines 12-18)
FW_PATH=$(grep "^framework_path:" "$PROJECT_ROOT/.framework.yaml" | sed 's/framework_path:[[:space:]]*//')
AUDIT_SCRIPT="$FW_PATH/agents/audit/audit.sh"
```

This correctly finds the audit script at `/usr/local/opt/agentic-fw/libexec/agents/audit/audit.sh`. **No bug here** — finding the script is correct.

### Link 2: Pre-push hook does NOT pass PROJECT_ROOT to the audit script

```bash
# .git/hooks/pre-push (line 37)
"$AUDIT_SCRIPT"
```

The hook calls the audit script **without setting PROJECT_ROOT**. The hook already has `PROJECT_ROOT` set correctly to the termlink project root (line 8: `PROJECT_ROOT="$(git rev-parse --show-toplevel)"`), but it doesn't export it or pass it as an env var to the audit script.

### Link 3: Audit script resolves PROJECT_ROOT from its own location

```bash
# audit.sh (lines 18-21)
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
FRAMEWORK_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"
PROJECT_ROOT="${PROJECT_ROOT:-$(git -C "$FRAMEWORK_ROOT" rev-parse --show-toplevel 2>/dev/null || echo "$FRAMEWORK_ROOT")}"
```

Since `PROJECT_ROOT` was not passed by the hook:
1. `SCRIPT_DIR` = `/usr/local/opt/agentic-fw/libexec/agents/audit`
2. `FRAMEWORK_ROOT` = `/usr/local/opt/agentic-fw/libexec`
3. `git -C "$FRAMEWORK_ROOT" rev-parse --show-toplevel` fails (brew install is not a git repo) → falls back to `FRAMEWORK_ROOT`
4. `PROJECT_ROOT` = `/usr/local/opt/agentic-fw/libexec` ← **THE BUG**

The audit script audits the framework install directory instead of the termlink project.

### Link 4: Stale framework_path (secondary issue)

`.framework.yaml` had `framework_path: /usr/local/Cellar/fw/1.1.0/libexec` which is a versioned brew path that breaks on upgrades. Fixed to `/usr/local/opt/agentic-fw/libexec` (the stable symlink) in this session, but this is a recurrence risk for all projects.

## Root Cause Summary

**The pre-push hook doesn't export PROJECT_ROOT to the audit script.** The audit script's fallback resolution derives PROJECT_ROOT from its own file location, which for brew-installed frameworks points to the framework install directory, not the project being pushed.

## Fix

### Immediate fix (termlink pre-push hook)

Add `PROJECT_ROOT="$PROJECT_ROOT"` before the audit script call:

```bash
# Before (broken):
"$AUDIT_SCRIPT"

# After (fixed):
PROJECT_ROOT="$PROJECT_ROOT" "$AUDIT_SCRIPT"
```

### Framework fix (for all projects)

The framework's `git.sh install-hooks` must generate pre-push hooks that pass PROJECT_ROOT. This affects all projects using shared framework installs.

**File to fix:** `/usr/local/opt/agentic-fw/libexec/agents/git/git.sh` — the `install-hooks` function that generates the pre-push hook template.

### Secondary fix: framework_path stability

`.framework.yaml` should use the brew stable symlink (`/usr/local/opt/agentic-fw/libexec`) not the versioned Cellar path. The `fw init` command should generate the stable path.

**File to fix:** `/usr/local/opt/agentic-fw/libexec/bin/fw` or the init logic that writes `.framework.yaml`.

## Remediation Instructions for Framework Agent

### Investigation Tasks

1. **Search for all hooks generated by `git.sh`** — Check if `commit-msg`, `pre-commit`, and other hooks have the same PROJECT_ROOT passthrough bug:
   ```bash
   grep -A5 "AUDIT_SCRIPT\|audit.sh" /usr/local/opt/agentic-fw/libexec/agents/git/git.sh
   ```

2. **Check `fw init` path generation** — Does it write the stable symlink or the Cellar path?
   ```bash
   grep -n "framework_path" /usr/local/opt/agentic-fw/libexec/bin/fw
   ```

3. **Audit all env var passthrough in hooks** — Any hook that calls a framework script needs to pass PROJECT_ROOT:
   ```bash
   ls /usr/local/opt/agentic-fw/libexec/agents/git/templates/
   grep -r "PROJECT_ROOT" /usr/local/opt/agentic-fw/libexec/agents/git/
   ```

4. **Check all existing projects** — Other projects using the framework may have the same broken pre-push:
   ```bash
   find ~/001-projects -name ".framework.yaml" -exec grep "framework_path" {} +
   find ~/001-projects -name "pre-push" -path "*/.git/hooks/*" -exec grep -L "PROJECT_ROOT.*AUDIT" {} +
   ```

### Remediation Steps

1. Fix the pre-push hook template in `git.sh` to include `PROJECT_ROOT="$PROJECT_ROOT"` before the audit call
2. Fix `fw init` to write the stable brew symlink path
3. Add a re-install hook command: `fw git install-hooks --upgrade` for existing projects
4. Add a regression test: pre-push hook in a shared-framework project must audit the project, not the framework
5. Register this as a gap in `gaps.yaml` with pattern: "hook-env-passthrough"

### Impact Assessment

- **Severity:** High — pre-push audit gate is silently auditing the wrong project, giving false confidence
- **Scope:** All projects using brew-installed framework with `fw git install-hooks`
- **Since:** Likely since the brew install mechanism was introduced
- **Detection gap:** No test validates that hooks audit the correct project root
