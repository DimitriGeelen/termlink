# T-2207: Add /closeable skill + scripts/list-closeable.sh — backlog-drain enabler

## Context

`fw task verify` surfaces tasks awaiting Human-AC click (69 today). The symmetric agent-side question — "which agent-owned tasks have 0 unchecked Agent ACs and are ready to close to partial-complete or full-complete RIGHT NOW?" — has no canonical verb. Operators (and agents) hand-roll python loops over `.tasks/active/` every session to find them. This task ships the canonical verb so backlog drain becomes one-keystroke.

## Acceptance Criteria

### Agent
- [x] `scripts/list-closeable.sh` exists, executable, with `--help` showing usage + flags
- [x] Walks `.tasks/active/`, classifies each task: `owner` (agent/human), Agent AC counts (checked/unchecked), Human AC counts (checked/unchecked), `status`, `horizon`, `workflow_type`
- [x] Filters to: `owner=agent` AND `status` in {captured, started-work, issues} AND `workflow_type != inception` AND `horizon != later` AND Agent unchecked = 0
- [x] Renders two sections: **Full-close-ready** (Human unchecked = 0 → fully closeable) and **Partial-complete-ready** (Human unchecked > 0 → close to partial-complete pending Human click)
- [x] Default human-readable table format (id, name, Agent ✓ count, Human ✗ count if partial, suggested command)
- [x] `--json` emits `{ok, full_close_ready:[...], partial_complete_ready:[...]}` envelope
- [x] Skill `/closeable` wrapping the script with empty-state messaging + next-step hints
- [x] Empty result path renders affirmative confirmation ("no closeable tasks right now — backlog is drained or fully blocked on humans") with `/peers` + `fw task verify` pointers
- [x] Self-test: running `bash scripts/list-closeable.sh --json` on current tree returns valid JSON; running without flags exits 0

# Pipefail/SIGPIPE hint (L-387): P-011 runs each command under `set -eo pipefail`.
# `cmd | grep -q PATTERN` exits 141 (SIGPIPE) when grep matches and closes stdin
# while the upstream is still writing — verification then "fails" even though
# the pattern was present. Safe pattern: capture first, grep the capture:
#     out=$(cmd 2>&1); echo "$out" | grep -q "PATTERN"
# Or:
#     cmd > /tmp/.out 2>&1 && grep -q "PATTERN" /tmp/.out
# Origin: L-387, captured 4× (T-1716, T-1838, T-1862, T-1863) before this hint.
#
# Single pipe only — no intermediate tail/awk/sed stages between capture and grep
# (T-2090): `echo "$out" | tail -3 | grep -q PAT` re-introduces the SIGPIPE risk
# the capture step closed off — the middle stage is what `grep -q` slams its
# stdin on. `echo "$out"` is small and immediate; grep scans the whole captured
# string anyway, so the tail-3 was cosmetic. Drop it: `echo "$out" | grep -q PAT`.
