## Acceptance Criteria

### Agent
- [x] `check_binary_freshness` downgrades the staleness WARN to PASS when it can PROVE `crates/` is unchanged between the installed binary version and HEAD (same major.minor, positive patch delta, `HEAD~delta` resolvable, no `crates/` commits in that range).
- [x] Any uncertainty keeps the WARN (fail-safe): differing major/minor, unparseable patch, non-positive delta, `HEAD~delta` not resolvable (shallow clone), or any `crates/` change in range — never silences a genuinely stale binary.
- [x] Live run on current repo state (crates/ unchanged for the commits since the installed binary) reports the binary check as PASS and substrate-preflight exits 0 (was WARN/exit 1).
- [x] Negative proof: a PATH-shimmed `termlink` reporting an old major.minor (e.g. 0.10.0) still WARNs (exit 1) — the downgrade does not leak to genuine cross-release staleness.
- [x] The PASS message names the distinction ("version drift only — no crates/ change since binary; rebuild not required") so an operator understands why no action is needed.

# Pipefail/SIGPIPE hint (L-387): P-011 runs each command under `set -eo pipefail`.
# `cmd | grep -q PATTERN` exits 141 (SIGPIPE) when grep matches and closes stdin
# while the upstream is still writing — verification then "fails" even though
# the pattern was present. Safe pattern: capture first, grep the capture:
#     out=$(cmd 2>&1); echo "$out" | grep -q "PATTERN"
# Or:
#     cmd > /tmp/.out 2>&1 && grep -q "PATTERN" /tmp/.out
# Origin: L-387, captured 4× (T-1716, T-1838, T-1862, T-1863) before this hint.
#
# Single pipe only — no intermediate tail/awk/sed stages between capture and grep
# (T-2090): `echo "$out" | tail -3 | grep -q PAT` re-introduces the SIGPIPE risk
# the capture step closed off — the middle stage is what `grep -q` slams its
# stdin on. `echo "$out"` is small and immediate; grep scans the whole captured
# string anyway, so the tail-3 was cosmetic. Drop it: `echo "$out" | grep -q PAT`.
