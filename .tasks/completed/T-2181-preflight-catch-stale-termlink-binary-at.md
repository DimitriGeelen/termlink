# T-2181: preflight: catch stale termlink binary at deploy time (T-2180 follow-up)

## Context

Discovered this turn during cold-start trio smoke (T-2180 close-time):
operator had `termlink 0.11.949` (8 days stale) while VERSION file showed
`0.11.1230`. The catalog at CLAUDE.md documents T-2076 `--only-stuck` and
T-2062 `fleet governor-status` — both shipped after 0.11.949 — so the
operator running `/substrate` per the cold-start trio recipe hit `unknown
flag`/`unrecognized subcommand` instead of the expected substrate digest.

This is a structural blind spot in `/preflight`: the existing 3 checks
(runtime_dir / hubs.toml / be-reachable) catch deploy-time misconfig but
not deploy-time staleness. Add Check 4: compare `termlink --version`
output against the project root `VERSION` file. WARN (not FAIL) when
the binary is older — substrate still works for primitives the binary
has, but newer documented features will surface as confusing errors.

## Acceptance Criteria

### Agent
- [x] `substrate-preflight.sh` gains `check_binary_freshness` that compares `termlink --version` to project root `VERSION`.
- [x] When `termlink` not on PATH: emits WARN (medium severity) with remediation pointing at install path.
- [x] When binary >= VERSION: emits PASS.
- [x] When binary < VERSION: emits WARN (medium severity) with remediation `cargo build --release && install -m 755 target/release/termlink ~/.cargo/bin/`.
- [x] When VERSION file unreadable: skip the check (don't emit anything) — no signal vs noise from foreign trees.
- [x] Header docstring + `usage()` block updated to document Check 4.
- [x] Smoke against current host (termlink 0.11.1230 vs VERSION 0.11.1230+) renders PASS for the new check.

# Pipefail/SIGPIPE hint (L-387): P-011 runs each command under `set -eo pipefail`.
# `cmd | grep -q PATTERN` exits 141 (SIGPIPE) when grep matches and closes stdin
# while the upstream is still writing — verification then "fails" even though
# the pattern was present. Safe pattern: capture first, grep the capture:
#     out=$(cmd 2>&1); echo "$out" | grep -q "PATTERN"
# Or:
#     cmd > /tmp/.out 2>&1 && grep -q "PATTERN" /tmp/.out
# Origin: L-387, captured 4× (T-1716, T-1838, T-1862, T-1863) before this hint.
#
# Single pipe only — no intermediate tail/awk/sed stages between capture and grep
# (T-2090): `echo "$out" | tail -3 | grep -q PAT` re-introduces the SIGPIPE risk
# the capture step closed off — the middle stage is what `grep -q` slams its
# stdin on. `echo "$out"` is small and immediate; grep scans the whole captured
# string anyway, so the tail-3 was cosmetic. Drop it: `echo "$out" | grep -q PAT`.
#
