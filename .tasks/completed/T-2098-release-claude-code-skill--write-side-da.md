# T-2098: /release claude-code skill — WRITE-side daily verb for substrate #1 (release claim by id)

## Context

Substrate primitive #1 (T-2019/T-2042 channel.claim) has four operator lifecycle verbs:
acquire, release, transfer, observe. Prior sessions shipped `/claims` (T-2093 READ) and
`/claim` (T-2097 acquire). The matching release verb still requires the operator to
remember `termlink channel release <claim-id>` with positional argument. This skill
completes the substrate #1 lifecycle at the skill tier so the daily-verb surface is
symmetric: acquire (`/claim`) → release (`/release`). Without it, operators who fluently
use `/claim` drop to CLI verb-recall friction at the end of every work unit.

Pair design: WRITE-side, mutates state. Loud-refusal taxonomy for known error classes
(CLAIM_NOT_FOUND, CLAIM_NOT_OWNED, AUTH_FAIL, RATE_LIMITED). Auto-resolves `--by`
identity from the same chain `/claim` uses (T-1857 sender-resolution: explicit flag →
env → `~/.termlink/be-reachable.state` → refuse). Never invents an owner.

Closes substrate primitive #1 WRITE-side surface at the skill tier (excluding transfer
which is its own daily-verb candidate — T-2099).
