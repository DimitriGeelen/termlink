# T-2070: fleet governor-status --only-pressured filter (show only hubs needing attention)

## Context

Operator UX: as the fleet grows, `fleet governor-status` output gets long.
For day-to-day "which hubs need my attention?" triage, an operator wants a
filter that returns only the hubs in trouble (unreachable, at capacity, or
rate-limited). This task adds `--only-pressured` as a presentation-level
filter on the already-shipped fleet aggregation. Pure presentation; no new
RPC; no schema change. Mirror of `fleet verify --exit-on-drift-only`
(T-1661) which gives the same "alert only on the bad subset" affordance
for rotation events.

## Acceptance Criteria

### Agent
- [x] `termlink fleet governor-status --only-pressured` flag exists.
- [x] Human-format output renders only hubs matching the pressured predicate (any of: unreachable / capacity_hits_total > 0 / rate_hits_total > 0 / active >= max); when zero hubs match, prints `All hubs healthy (0/N pressured)` line with the visible totals.
- [x] JSON output respects the filter: only the matching hubs appear in `hubs[]`; the summary footer keeps the fleet-wide totals (so the operator sees "1/5 pressured" with the original counts) and gains a `shown` field equal to `hubs[].len()`.
- [x] Pure predicate helper `governor_hub_is_pressured(...)` extracted and ≥2 unit tests cover (a) capacity_hits > 0 case (b) rate_hits > 0 case (c) at-capacity (active >= max) case (d) unreachable case (e) clean fleet (predicate returns false).
- [x] `cargo build -p termlink --release` succeeds.
- [x] CLAUDE.md BACKPRESSURE row mentions the new filter flag.
