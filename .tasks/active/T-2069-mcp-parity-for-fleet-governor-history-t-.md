# T-2069: MCP parity for fleet governor-history (T-2068 agent-callable companion)

## Context

Wraps T-2068's `fleet governor-history` CLI verb as a
`termlink_fleet_governor_history` MCP tool. Agent-callable parity for the
read-side companion to Track G's audit trail — answers "has this hub been
backpressured / rate-limited in the last N days?" without shelling out.
Mirror of T-1687 (MCP parity for T-1671 `fleet history`). Read-only file
scan of `~/.termlink/governor.log`.

## Acceptance Criteria

### Agent
- [x] `termlink_fleet_governor_history` MCP tool exists, is registered in the tool list in tools.rs and is callable.
- [x] Parameters: `since_days` (Option<u32>, default 7, clamped 1..=365), `hub` (Option<String>), `log_path` (Option<String>, override default `$HOME/.termlink/governor.log`). Out-of-range `since_days` returns a structured error string (not a panic).
- [x] Returns `{ok, entries, summary{total, per_hub, since_days, hub_filter, malformed_lines_skipped, log_path}}` JSON. Empty/missing log returns `{ok: true, entries: [], summary: ..., hint: "no governor history yet — run `fleet governor-status --watch --log <path>` to start capturing"}`.
- [x] Per-hub aggregate in `summary.per_hub` includes `events`, `cap_hits_total`, `rate_hits_total`, `dedupe_hits_total` per hub — symmetric to the CLI `--json` summary shape (T-2068).
- [x] Unit test asserts the empty-log envelope shape; a populated-log test parses a temp log fixture and validates the `per_hub` aggregate.
- [x] `cargo build -p termlink-mcp --release` succeeds; `cargo test -p termlink-mcp --release --lib fleet_governor_history` passes.
- [x] CLAUDE.md BACKPRESSURE row mentions the new MCP tool alongside the CLI verb.
