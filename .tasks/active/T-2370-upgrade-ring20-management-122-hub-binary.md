# T-2370: Upgrade ring20-management (.122) hub binary 0.11.296 -> >=0.11.324 and restart through supervision (clears fleet-binary canary)

## Context

The fleet-binary canary (G-069) fires because ring20-management (.122) serves
hub_version 0.11.296, below its declared floor 0.11.324 in
`.context/cron/fleet-version-floors.conf`. Live-confirmed twice via
`termlink fleet doctor --json` (address 192.168.10.122:9100). Remediation is a
remote binary deploy to the .122 container + a supervised hub restart
(G-070: through the unit / watchdog, not a detached relaunch; PL-104). Must
watch for PL-021 (volatile /tmp → secret+cert rotation) on restart per
CLAUDE.md §"Special case — volatile runtime_dir" (T-1294 confirmed .122 uses
ring20-watchdog.sh with an explicit runtime_dir off /tmp).
