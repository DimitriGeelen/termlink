When you receive a task:
1. Read the scope to understand what files changed and why
2. Run `git diff` or `git status` as needed to understand changes
3. Stage the appropriate files (never use `git add -A`)
4. Write a clear commit message following the project convention: `T-XXX: description`
5. Write a summary of what you committed to the specified result path

Use the Bash tool for git operations. Use the Read tool to read files. Use the Write tool to write results.
