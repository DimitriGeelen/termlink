## Post

Over 25 years of working on complex IT programmes, I arrived at a principle I now believe is universal:

Effective intelligent action — whether by a person, a team, or an AI agent — requires five things:

1. Clear direction
2. Awareness of context
3. Awareness of resource constraints
4. Awareness of downstream impact
5. Engaged, capable actors

Remove any one and the system degrades.

At Shell, I built a governance framework for IT transitions based on this principle — quality gates, assurance areas, structured handovers. Shell adopted it as the global standard. It has been used for over 1,000 transitions worldwide.

When I started building with AI coding agents (Claude Code, Cursor, Copilot), I recognised the same failure modes:

- No traceability — files change with no record of why
- No memory — every session starts from zero
- No risk awareness — destructive commands execute without structured approval
- No learning — the same mistakes recur because nothing captures what went wrong

So I applied the same governance principle to a new domain.

The Agentic Engineering Framework adds structural governance to AI coding agents:

→ Task-first enforcement — file edits are blocked unless an active task exists
→ Three-layer memory — decisions, patterns, and task histories persist across sessions
→ Tiered approval — destructive commands require human sign-off
→ Context budget management — auto-handover before the agent loses coherence
→ Continuous audit — 130+ compliance checks, every 30 minutes
→ Healing loop — failures become institutional knowledge

I used the framework to build the framework. 500+ tasks completed. 98% commit traceability. Every architectural decision recorded with rationale.

The domain changed. The principle did not.

Open source (Apache 2.0): https://github.com/DimitriGeelen/agentic-engineering-framework
