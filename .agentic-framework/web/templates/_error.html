Your form submission session expired

This happens when your browser holds an old session cookie — usually after the Watchtower server restarted, the browser was offline for a long stretch, or strict cookie policy is in effect for this origin. Nothing was recorded; you can safely retry.

Recover in 10 seconds

  1. {% if back_url %} Go back to the form {% else %} Navigate back to the form you were submitting {% endif %} and press Cmd-R (macOS) or F5 (Linux/Windows) to refresh — the form re-renders with a fresh CSRF token.
  2. Re-submit your action. It should land this time.

If refresh doesn't fix it

Clear cookies for {{ request.host }} in your browser's site settings, then revisit the form. The fresh session will issue a new cookie.

CLI fallback (always works)

Every Watchtower action has a fw command equivalent. For inception decisions: 

cd /opt/termlink && .agentic-framework/bin/fw inception decide T-XXX go|no-go|defer --rationale "..."

For task AC ticks, use fw task verify + edit the task file directly. See CLAUDE.md Quick Reference table for the full verb list.
